# Patch Notes

每次向 GitHub 推送代码前，都必须在此文件中补充本次推送的用户可见修复、行为变化、重要实现和验证结果。

## 2026-07-24 - 项目根目录自动恢复

### 修复内容

- 项目数据库新增稳定项目身份元数据；新建、打开、迁移和恢复项目时会写入或校验该身份，旧项目在目录仍可访问时自动回填。
- Web 刷新项目数据时，如果注册的根目录已经不存在，会在配置的允许项目目录内查找对应的 `.project-context/project.db`，无需再次手动输入新路径。
- 自动恢复只接受项目身份精确匹配的唯一候选；找不到、身份错误、目录冲突或存在多个副本时不会猜测，继续保留“编辑项目”手动处理入口。
- 自动发现只遍历允许目录，跳过版本控制目录、依赖和构建输出，并对候选数据库执行完整性、结构版本和项目身份校验。
- 默认项目名称会随根目录名称一起更新；用户自定义的显示名称保持不变。
- 自动恢复后重新绑定活跃文件 watcher，并保留原有防抖配置。

### 验证结果

- `npm run typecheck`：通过。
- `npm test -- --run tests/personal-storage.test.ts tests/ui.test.ts`：14 个测试通过。
- `npm test -- --run tests/personal-storage.test.ts`：新增回填测试后 12 个测试通过。
- `npm test`：52 个测试全部通过。
- `npm run build`：通过。

### 使用说明

在允许项目目录范围内手动重命名或移动项目根目录后，刷新 Web 页面即可自动同步新路径。若移动到了未授权目录、复制出了多个带相同身份的项目数据库，或数据库尚未写入稳定身份，系统会保持原记录不变，此时可使用“编辑项目”明确指定目录。

## 2026-07-24 - 项目目录重命名同步修复

### 修复内容

- 在 Web 项目画像中新增“编辑项目”入口，可同时更新项目显示名称和根目录。
- 保存项目资料后，同步刷新任务流水线、项目画像、规则和上下文中的项目选择数据。
- 项目根目录迁移后，将活跃文件 watcher 重新绑定到新目录，并保留原有防抖配置。
- 新根目录继续使用现有真实路径、允许目录和项目冲突校验，避免越权或错误指向。
- 移动端项目操作区调整为两列布局，保证新增操作不会产生溢出。

### 验证结果

- `npm run typecheck`：通过。
- `npm test -- --run tests/ui.test.ts tests/personal-storage.test.ts`：11 个测试通过。
- `npm test`：48 个测试全部通过。
- `npm run build`：通过。
- Playwright 桌面端和 `390x844` 移动端验证通过，浏览器控制台无错误或警告。

### 使用说明

外部重命名项目目录后，在 Web 的“项目画像”中选择项目，点击“编辑项目”，填写新的项目名称和已存在的新根目录并保存。系统无法仅根据失效的旧路径通用推断任意新路径，因此外部改名后需要明确提交一次新目录。
